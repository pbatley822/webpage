C.L.I.P.P.

As part of my C++ class, we were given this file that comprised of several numbers in a text document. When processed through an image program like Adobe PS or GIMP, it can be interpreted as an image. This particular kind of file, referred to as a .PGM file, has 5 elements: a file format("P2"); a comment(denoted by a '#'); a resolution(two numbers: columns and rows); a maximum grey value(applies to each pixel); and the pixels represented by individual integers. Wiki entry about file formats

With this knowledge about this and other file formats, we were tasked to perform several operations on these kinds of images: Rotate, resize, pixilize(distort), dilate and erode(morphological operations) and others. With this in mind, a logical path had to take place in order for me to solve the problems at hand. I had several steps to take to complete the task.

After many laborious hours the first assignment we were given was completed. The concept seemed simple enough, but debugging the program turned out to be the biggest obstacle in my way. Small mistakes, many beginning programmers make, were plaguing me. We were given many more assignments throughout the semester and I continued to learn from my mistakes. Through these torments and tribulations, came my strong and confident programming skills that I posses today.

These projects have taught me a lot about how to solve a problem logically, even if the problem wasn't necessarily one that was set before you initially. I believe the process of learning, programming skills or otherwise, is different for every one. I benefit from diving head first into the work and immersing myself with the task. In this regard, I have become a strong and capable learner.
